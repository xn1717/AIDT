主張辨識
為什麼選擇 S1 學生？
選擇理由：權力不對稱
學生是唯一無法拒絕參與、無法撤回授權資料的行動者。教師可選擇教學方式、家長可選擇學校、企業可決定是否參與——學生面臨的是「系統強制」，是最脆弱的倫理主體。

不應成為實驗對象
未成年人享有特別保護的道德地位，不應被視為資料採集對象或演算法實驗的被測試者

純 Is，無規範詞
P2
規範前提 — 義務論 ＋ 分配正義
未成年人享有不可因教育承諾而削弱的隱私保護義務（《兒童權利公約》第 3、16 條）。當弱勢學生只承擔風險卻無法獲益，系統設計本身構成分配不正義，責任應由部署方承擔。
義務論 分配正義
C
結論
韓國教育部在未建立 （1）知情同意機制、（2）替代學習方案、（3）獨立資安稽核 前，不應強制部署 AIDT 並蒐集學生細粒度行為資料。系統造成傷害時，教育部與開發商應承擔主要賠償責任。
反論識別與駁斥
來自 S4 企業的反論
最強版反論
「企業是被動配合方，不應承擔連帶責任」
企業在政府強制時程與模糊審查標準下投入資源，屬制度性壓迫下的被動配合。要求企業連帶課責，將形成寒蟬效應，使公共教育科技市場萎縮，最終受害的是學生。
保護商業可預期性 中小型開發商利益
此反論值得認真對待，因為...
它保護了教育科技產業的中小型開發商的商業可預期性與制度公平性利益。若採納我們的論證，可能形成寒蟬效應，使公共教育科技市場萎縮。
駁斥：為何反論仍無法成立
政治壓力 ≠ 專業判斷義務的消失
企業具備獨立評估資安風險的專業能力，這正是其參與競標的前提資格。當開發時程不足以確保資安品質，其義務是主動揭露風險或拒絕交付——「執行命令」在醫療、工程倫理中從未被接受為免責理由。
德性論
反論的自我矛盾
反論擔憂的「寒蟬效應」恰好說明了相反結論：若企業可在不合理條件下交付缺陷系統並免責，這才是獎勵「配合換合約」而非「誠信換信任」的危險誘因結構。
自身立場限定
何種情境下成立或需修正？
完全成立的情境
強制普及 ＋ 無法退出
當 AIDT 被強制納入正規課程、學生無法選擇退出，且資料蒐集涵蓋細粒度行為追蹤時，義務論的保護義務具有最高優先性。任何以教育效益或產業發展為由的豁免主張均不能成立。
義務論優先 不可豁免
需要限定的情境
自願試驗 ＋ 充分保障
若情境轉為自願參與的有限試驗計畫，且同時滿足：學生與家長可隨時退出、替代方案存在、資料蒐集符合最小必要原則，則有限度的試驗性部署在倫理上具有可辯護空間。
比例責任 效益可納入權衡
限定的本質
此限定並非對原論證的退讓，而是對「強制普及」與「知情自願試驗」在倫理性質上的精確區分：前者觸動的是不可讓渡的基本權利，後者在充分保障的前提下可納入效益權衡。

強化論證的觀點
「標準學習者」預設的代價
行為模式異常的弱勢學生（新移民、學習障礙生）會被系統標記為「低能力」，固化本應被打破的偏見。教師的典型反應是「不知道如何向家長解釋」——知情同意在現場形同虛設。
強化 P1 強化 C
挑戰論證的觀點
「完全不用不見得更好」
若系統輸出由受訓教師詮釋與調節，而非直接作為評量依據，部分風險是可以被管理的。這提示論證需處理「資料詮釋權歸屬」，不只是限制蒐集。
輕微挑戰
BRiCS 修正 — S3 教師欄補充第四項條件
應在原有三項前提條件之外補充：建立教師資料素養培訓機制，使教師具備詮釋、質疑與必要時否決系統輸出結論的專業能力。論證從「限制資料蒐集」延伸至「確保人類判斷不被系統取代」。
